
git を勉強する順番

1) ひとりで git を使う(branchはmainのみ)

2) branch を使った開発

3) チームで開発する



git のリポジトリの考え方

1) リモート・リポジトリ

2) ローカル・リポジトリ

3) ソース・コード群



コマンド・プロンプトのコマンド

dir -- ファイル・ディレクトリの一覧

cd -- 現在位置の表示

cd <移動先> -- 位置の移動

TABキーによるファイル名・ディレクトリ名の補完

↑↓キーによるヒストリ機能



git のコマンド

git clone https://github.com/<ユーザー名>/<リポジトリ名>.git

git fetch -- リモートの変更をとりこむ

git status -- 現在の状態を表示

git add . -- 変更箇所を追加

git commit -m "修正したところなどをメッセージとして残す"

git push -u origin main -- リモートに反映させる

git log -- 現在までの記録を表示



git は アクセストークンによる認証

git config --global credential.helper store
  -- いちいちアクセストークンを入力しなくてもすむようにする
